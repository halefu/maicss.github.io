ubuntu环境安装笔记

因为有好几个平台，虚拟机啊，物理机，还有一个mac，经常安装很多次环境，但是又一直忘，就整理一次好了。

ubuntu

su用户

超级用户设置。sudo passwd [root]，然后键入两次密码就行。[root]可能不是必须的

安装java8

因为安装了三次（我比较折腾），每次都要配置很多环境，而我又是新手，每次都记不住，很苦恼。有次偶然发现了一个不错的一键安装式的，就是webupd8。只需要四个命令就行了。

敲完之后，就直接在线安装了，这个是图，忽略我那渣速度。
![img]
因为ubuntu默认安装了一个openjdk什么的，还要覆盖环境变量设置。但是这个方式安装是直接设置好环境变量。
懒人必备啊，现在我还不知道这个是怎么直接在线安装的，以后再说吧。

安装pycharm

走到这一步你会发现，这个都是按照顺序来的。唉都是泪，一点点坑爬过来的。
因为这个要求java编译安装什么的…。

先给他们打打广告。他们家的编辑器真的用心啊，真的。买三年的话，一年大约300块，不算很贵的，有能力的话支持一下。

安装完java之后，下载压缩包，然后tar -zxvf解压，进入解压后的目录，然后是bin文件夹，之后直接执行./pycharm.sh即可自动安装。

安装zsh

因为是刚入门，现在还体会不到zsh和bash的区别，最明显的就是在git的文件夹下面会出现相关的提示信息吧。大家都推崇的文件夹跳转的jump插件，因为需要跳转的文件夹比较少，还体会不到这种方便，这种会在多次经常而且很深的目录之间切换会很爽吧。

  • 安装直接sudo apt-get install zsh就搞定了。
  • 看一下配置里面有没有zsh：cat /etc/shells。
  • 然后切换默认的shell：chsh -s /usr/bin/zsh，文件路径是我自己的，别照抄就行。但是我在ubuntu下试过两次，这个文件夹都可以的。

我被震惊了

没事在逛网页的时候，发现了这个

Zsh的最初版本是在1990年由Paul Falstad写成，当时他是普林斯顿大学的学生。他以普林斯頓大學助理教授邵中(Zhong Shao)的名字來命名zsh。[来源：维基百科]

然后我去看了看这个人。

这个是科技大给出的简介，这个是yale给出的简介。

我被震惊了。

再看看科技大的第一行的那三个人，我真的被吓到了。

升级的时候，老是说我空间不足，使用sudo apt-get autoremove或者使用sudo apt-get clean腾出来一点空间。

nano 常用快捷键

另外这玩意不知道怎么才能设置一个tab为4个空格，好烦（对，我就是tab变空格党，来打我啊^_^）

C => Ctrl, A => Alt

  • C+O => 保存
  • C+X => 退出编辑器
  • A+\ => 跳转到文件头
  • A+/ => 跳转到文件尾
  • A+W => 查找
  • A+R => 替换
  • C+_ => 跳转到行
  • C+K => 剪切当前行 / 开关：A+K
  • C+U => 粘贴
  • A+M => 鼠标支持
  • A+$ => 软换行(这个好扯, mac上的这个是截屏)
  • A+] => 跳转到对应括号
