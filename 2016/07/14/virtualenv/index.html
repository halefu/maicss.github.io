virtualenv

这个是python的虚拟环境的教程。

为什么要使用虚拟环境

好像一个长时间写代码的人都喜欢环境或者作用域的隔离。
当我们安装好python之后，各种pip install。这个安装都是全局下的安装，但是当项目大的时候，就会遇到各种特殊情况。

比如我就是想要某个版本的，你该说啦，我安装的时候指定安装版本还不行吗，这个是可以的。
但是还有更变态的需求，我想测试一下两个版本都怎么样。遇到这个需求就SB了不是吗。

一个健全的代码要满足所有的需求的（说起来都是泪）。

于是就有了虚拟环境。就是一个独立的作用域。一个你可以在里面安装各种东西，而全局下的还是干净的玩意。
你可以有多少个需求就做多少个虚拟环境，不想用的时候直接rm -rf就完事了。

而且一直使用全局安装的话，还会遇到安全问题。
因为pypi是不检测用户提交包的，而且谁都可以往上面提交包（chrome也是这德行，Firefox就有检测机制，但是Firefox好卡…）。所以，如果哪天你安装包的时候，不小心少敲了一个字母或者多敲了一个字母，或者写成其他的了，而且很巧pypi上面正好有一个盗版的包，你可是用sudo安装的哦，好怕怕！！

So , virtualenv is the Prefect!

下面我们来说说这玩意怎么具体实现。

这玩意的实现有两种方式，而这篇文章就是对比这两种方式。

这篇文章不是这两种方式的教程。

1，virtualenv

这个比较单纯。

Virtual Python Environment builder

使用pip全局安装之后，你就能使用virtualenv命令了。

因为单纯，所以纯洁（这是什么鬼）。

所以你可以使用virtualenv -p /your-python-env-dir your-env-name 比如：virtualenv -p python3 py3创建虚拟环境（python3是个软连接，不懂软连接的Google）。这样之后，你就会在当前文件夹创建一个py3的文件夹，然后里面有个初始化了的py3环境。
然后source py3/source/activate就可以激活这个虚拟环境，注意路径就好了。

-p是指定python版本的意思。一般情况下，机器上是默认安装了python2的，所以如果你安装了python3，就可以装两个不同的虚拟环境能满足日常需求了。~当然你也可以创建5000个py2的虚拟环境，5000个py3虚拟环境，就可以解锁python技能了~

2，conda

这个比较重量级。是个集成环境。

Conda is an open source package management system and environment management system for installing multiple versions of software packages and their dependencies and switching easily between them. It works on Linux, OS X and Windows, and was created for Python programs but can package and distribute any software.
